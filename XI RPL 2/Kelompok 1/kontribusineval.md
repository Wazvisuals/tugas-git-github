Penjelasan git revert

Apa itu Git Revert?.
Git Revert adalah perintah Git yang memungkinkan kamu membatalkan perubahan yang sudah dilakukan pada kode sumber.
Ini dilakukan dengan membuat commit baru yang membatalkan perubahan sebelumnya.
Jadi, commit sebelumnya tidak dihapus, tapi ada commit baru yang membatalkan perubahannya. .

Kegunaan Git Revert
1. Membatalkan Perubahan: Git Revert memungkinkan kamu membatalkan perubahan yang sudah dilakukan tanpa menghapus commit sebelumnya. 
2. Mengembalikan Kode ke Versi Sebelumnya: Git Revert memungkinkan kamu mengembalikan kode ke versi sebelumnya tanpa kehilangan riwayat perubahan.
3. Menghindari Konflik: Git Revert membantu menghindari konflik yang mungkin terjadi jika kamu menggunakan perintah git reset untuk membatalkan perubahan.
4. Mempertahankan Riwayat Perubahan: Git Revert mempertahankan riwayat perubahan kode sumber, sehingga kamu bisa melacak perubahan yang sudah dilakukan.

Cara Menggunakan Git Revert
1. git revert : Perintah ini membuat commit baru yang membatalkan perubahan yang dilakukan oleh commit dengan hash yang ditentukan.
2. git revert -n : Perintah ini membatalkan perubahan yang dilakukan oleh commit dengan hash yang ditentukan, tapi tidak membuat commit baru secara otomatis. 
Dengan Git Revert, kamu bisa membatalkan perubahan yang sudah dilakukan dengan aman dan efektif, sambil mempertahankan riwayat perubahan kode sumber.
